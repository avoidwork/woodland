# Fix for Issue #374: INT_1e6 ReferenceError in app.files()

## Root Cause
Rollup's mangler sees `INT_1e6` and interprets `1e6` as scientific notation, renaming it inconsistently across the bundle:
- Bundled `config.js` inlines as `INT_1e6$1` (correctly suffixed)
- Bundled `woodland.js` own reference stays as bare `INT_1e6` (never defined → ReferenceError)

## Files to Change

### 1. `src/constants.js` (line 95)
```
- export const INT_1e6 = 1e6;
+ export const INT_1000000 = 1e6;
```

### 2. `src/woodland.js` (line 563)
```
- const msValue = Number(diff / INT_1e6).toFixed(this.#digit);
+ const msValue = Number(diff / INT_1000000).toFixed(this.#digit);
```

### 3. `src/config.js` (line 16, 38)
```
- INT_1e6,
+ INT_1000000,

- bodyLimit: INT_10 * INT_1e6,
+ bodyLimit: INT_10 * INT_1000000,
```

### 4. `src/logger.js` (line 13, 213)
```
- INT_1e6,
+ INT_1000000,

- return TIME_MS.replace(TOKEN_N, Number(arg / INT_1e6).toFixed(digits));
+ return TIME_MS.replace(TOKEN_N, Number(arg / INT_1000000).toFixed(digits));
```

### 5. Rebuild
```bash
npm run build
```

### 6. Verify
```bash
npm test
```
